# москвещи — серверная демо-версия

Это серверный проект для сайта **«москвещи»**.

## Что уже есть

- регистрация и вход пользователей;
- пароли хранятся на сервере в виде хеша PBKDF2 + salt;
- пароль администратора не находится в HTML;
- товары хранятся на сервере;
- корзина хранится на сервере;
- заявки на пополнение баланса;
- баланс пополняет только администратор;
- активные заказы в админ-панели;
- изменение статусов заказов;
- блокировка пользователей;
- Telegram-уведомления через сервер;
- Telegram-токен хранится в `.env`, а не в браузере.

## Как запустить на компьютере

### 1. Установить Node.js

Нужно установить Node.js версии 18 или выше.

### 2. Открыть папку проекта в терминале

```bash
cd moskveshi-server
```

### 3. Создать файл `.env`

Скопируйте `.env.example` в `.env`.

На Windows можно просто скопировать файл вручную.

В `.env` обязательно поменяйте пароль администратора:

```env
ADMIN_EMAIL=admin@moskveshi.ru
ADMIN_PASSWORD=ВАШ_СЛОЖНЫЙ_ПАРОЛЬ
```

Telegram можно настроить позже:

```env
TELEGRAM_BOT_TOKEN=токен_бота
TELEGRAM_CHAT_ID=ваш_chat_id
```

### 4. Установить зависимости

```bash
npm install
```

### 5. Запустить сервер

```bash
npm start
```

После запуска откройте:

```text
http://localhost:3000
```

## Вход администратора

Администратор создается при первом запуске из `.env`:

```env
ADMIN_EMAIL=...
ADMIN_PASSWORD=...
```

Если вы уже запустили сервер и хотите поменять администратора, удалите файл:

```text
data/db.json
```

и запустите сервер заново.

## Где хранятся данные

В демо-версии данные хранятся в файле:

```text
data/db.json
```

Для настоящего сайта лучше заменить JSON-файл на PostgreSQL.

## Как опубликовать на Render

1. Создайте GitHub-репозиторий.
2. Загрузите туда этот проект.
3. На Render создайте **New Web Service**.
4. Подключите GitHub-репозиторий.
5. Укажите:

```text
Build Command: npm install
Start Command: npm start
```

6. В Environment Variables добавьте:

```env
ADMIN_EMAIL=...
ADMIN_PASSWORD=...
TELEGRAM_BOT_TOKEN=...
TELEGRAM_CHAT_ID=...
PAYMENT_CARD=2200 7021 1871 3952
CONTACT_TELEGRAM=@kadwss
SITE_NAME=москвещи
```

7. Нажмите Deploy.

## Важно

Это серверная демо-версия. Она уже намного безопаснее HTML-файла, но для настоящей коммерческой эксплуатации нужно добавить:

- PostgreSQL вместо JSON-файла;
- HTTPS-домен;
- восстановление пароля;
- подтверждение email;
- реальные платежи или автоматическую сверку переводов;
- резервное копирование базы;
- защиту от спама и перебора паролей.
